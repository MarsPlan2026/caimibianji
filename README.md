# 🎮 猜谜游戏编辑器 (Riddle Game Editor)

## 点击此[网址](https://marsplan2026.github.io/caimibianji)试用

一个**可视化编辑器**，用于快速创建多机制叠加的**猜谜解谜游戏**。支持 12+ 种折磨机制的任意组合，轻松制作复杂的网页谜题游戏。

## ✨ 核心特性

### 🎯 可视化关卡编辑
- 📝 直观的表单化界面，无需代码即可编辑关卡
- 🏷️ 支持关卡ID、提示信息、正确答案的自由配置
- ➕ 一键添加/删除关卡
- 💾 编辑完成后一键生成完整的 `.html` 游戏文件

### 🔄 反向导入解析
- 📂 将之前生成的游戏 `.html` 文件拖入编辑器
- 🔍 自动识别并还原所有关卡和已勾选的机制
- ✅ 100% 无损恢复，支持继续编辑

### 🎨 12+ 种折磨机制（可自由堆叠）

| 机制 | 效果 | 难度 |
|------|------|------|
| **大括号精准隐形** | 提示词中 `{}` 里的字变纯白 | ⭐ |
| **隐形墨水** | 输入框的字变纯白色 | ⭐ |
| **瞬移输入框** | 鼠标一碰输入框就跑路 | ⭐⭐ |
| **大写锁定** | 强制输入的字母全部变大写 | ⭐⭐ |
| **反向恶作剧** | 打字时文字从后往前倒序显示 | ⭐⭐ |
| **灵异干扰** | 每隔0.2秒自动往框里塞随机数 | ⭐⭐⭐ |
| **绝对静止** | 鼠标晃动就清空输入框 | ⭐⭐⭐ |
| **痛击提交** | 鼠标一摸提交按钮就秒清输入框 | ⭐⭐⭐ |
| **天翻地覆** | 整个页面每3秒颠倒旋转180度 | ⭐⭐⭐ |
| **空间压迫** | 回答错误时输入框变窄 | ⭐⭐ |
| **闪避按钮** | 提交按钮满屏幕乱飞 | ⭐⭐⭐ |
| **更多机制** | 持续更新中... | — |

**这些机制可以任意组合叠加！** 例如同时启用「反向+隐形+瞬移」会产生极限折磨效果。

## 📖 使用教程

### 第一步：创建关卡
1. 点击 **"+ 添加新关卡"** 按钮
2. 在表单中填入：
   - **关卡ID**：自定义数字 ID
   - **提示信息**：玩家看到的谜题提示
     - 💡 提示：用 `{隐形文本}` 语法可以隐藏关键线索
   - **正确答案**：游戏检验用的标准答案

### 第二步：选择折磨机制
在「赋予本关的折磨机制」区域勾选想要的机制：
- 可以同时勾选多个机制
- 机制会在游戏中同时生效并叠加
- 例：同时勾选「反向」和「隐形墨水」会导致输入被倒序显示且看不见

### 第三步：生成游戏
1. 编辑完所有关卡后，点击 **"一键打包并下载成品游戏"**
2. 自动生成一个独立的 `.html` 文件
3. 分享给朋友，让他们在浏览器中游玩

### 第四步：继续编辑（可选）
- 将已生成的 `.html` 游戏文件拖入编辑器的导入区
- 编辑器自动还原所有关卡和机制配置
- 修改后再次导出即可

## 🎛️ 高级功能

### 💾 自动保存
编辑器会将当前编辑的内容保存到浏览器本地存储，刷新页面也不会丢失

### 📥 JSON 导入/导出
- 导出配置为 JSON 文件便于备份和分享
- 从 JSON 文件导入配置快速恢复编辑

### 🔐 答案加密
导出的游戏中答案使用 Base64 编码，防止简单的"查看源代码"破解

### 🛡️ 反调试保护
生成的游戏包含反开发者工具代码，增加破解难度

### 👤 作者暗门
游戏中左下角有隐藏的作者暗门，输入关卡号可跳过关卡（方便测试）

## 📊 使用示例

### 创建一个三关的游戏

**第1关：简单题**
- 提示：`密码其实是 {BLACK}，仔细看！`
- 答案：`BLACK`
- 机制：大括号精准隐形 + 反向恶作剧
- 效果：玩家看不到提示中的 "BLACK"，且输入会倒序显示

**第2关：中等难度**
- 提示：`输入框好像坏了...`
- 答案：`HELP`
- 机制：隐形墨水 + 瞬移输入框 + 灵异干扰
- 效果：输入看不见、输入框乱跑、还有随机数自动塞入

**第3关：极限折磨**
- 提示：`这一关，我全开了。`
- 答案：`WIN`
- 机制：勾选全部 12 种机制
- 效果：完全无法正常输入，必须靠智慧和耐心

## 🐛 故障排除

### 问题：导入文件后显示"解析失败"
**原因**：文件不是由本编辑器生成的  
**解决**：确保导入的是 `.html` 格式的游戏文件

### 问题：生成的游戏打开是空白
**原因**：可能是浏览器缓存  
**解决**：按 `Ctrl+Shift+Delete` 清除缓存后重试

### 问题：机制在游戏中没有生效
**原因**：该机制的 JavaScript 初始化可能冲突  
**解决**：尝试调整机制的勾选顺序，或减少同时启用的机制数量
